# 🚀 ProProfit — Инструкция по развёртыванию

Этот файл — пошаговое руководство, как вывести сайт в интернет и включить общие отзывы.
Делаем в 3 этапа: **GitHub → Vercel → Supabase**. Каждый аккаунт бесплатный.

---

## ЭТАП 1. GitHub (хранилище кода)

1. Зайдите на **github.com**, нажмите **Sign up**.
2. Email: `unilive2025me@gmail.com` (или любой ваш), придумайте username (например `proprofit-team`) и пароль.
3. Подтвердите email кодом из письма.
4. Создайте репозиторий: кнопка **New** → имя `proprofit-website` → **Private** или **Public** → **Create repository**.
5. Скажите ассистенту: **username + название репозитория**. Для автозалива нужен **Personal Access Token**:
   - Профиль → **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)** → **Generate new token (classic)**
   - Срок: 90 дней, права: отметьте **repo** (весь блок)
   - Скопируйте токен (он показывается один раз) и передайте ассистенту.

> 🔒 **Безопасность токена:** токен даёт доступ только к вашим репозиториям с кодом сайта. Он НЕ даёт доступ к почте, деньгам, паролям. Его можно отозвать в любой момент в тех же настройках. Это стандартный способ.

---

## ЭТАП 2. Vercel (хостинг — сайт в интернете)

1. Зайдите на **vercel.com** → **Sign Up** → выберите **Continue with GitHub** (вход в один клик).
2. Разрешите доступ Vercel к вашему GitHub.
3. **Add New → Project** → выберите репозиторий `proprofit-website` → **Import**.
4. Framework Preset: **Other** (это статический сайт). Root Directory: оставить как есть.
5. **Deploy**. Через ~1 минуту сайт будет доступен по адресу вида `profit.team`.
6. Дальше: каждое изменение кода в GitHub автоматически обновляет сайт. Ассистент пушит правки — сайт обновляется сам.

### Свой домен (опционально, ~$10/год)
- Купите домен (например на **Namecheap**, **Reg.ru**, **Porkbun**): `proprofit.team`, `proprofit.site` и т.п.
- В Vercel: проект → **Settings → Domains → Add** → введите домен → следуйте инструкции (прописать DNS у регистратора).

---

## ЭТАП 3. Supabase (общие отзывы и тикеты)

Без этого этапа отзывы работают локально (видны только автору). С ним — видны всем.

1. Зайдите на **supabase.com** → **Start your project** → войдите через GitHub.
2. **New project**: имя `proprofit`, придумайте пароль БД (сохраните), регион — ближе к аудитории (например Frankfurt). **Create**.
3. Когда проект создастся: слева **SQL Editor** → **New query** → вставьте и запустите SQL ниже (создаёт таблицы):

```sql
-- Таблица отзывов
create table reviews (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  project text not null,
  name text,
  rating int check (rating between 1 and 5),
  text text not null,
  photo text,
  video text
);

-- Таблица тикетов (обращения с формы контактов)
create table tickets (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  name text,
  contact text,
  topic text,
  message text
);

-- Включаем защиту строк
alter table reviews enable row level security;
alter table tickets enable row level security;

-- Разрешаем всем читать отзывы
create policy "public read reviews" on reviews for select using (true);
-- Разрешаем всем добавлять отзывы
create policy "public insert reviews" on reviews for insert with check (true);
-- Тикеты: разрешаем всем добавлять (чтение — только из админки Supabase)
create policy "public insert tickets" on tickets for insert with check (true);
```

4. Слева **Project Settings → API**. Скопируйте:
   - **Project URL** (вид `https://xxxx.supabase.co`)
   - **anon public** ключ (длинная строка)
5. Передайте ассистенту эти два значения — он впишет их в `assets/config.js`, и общие отзывы заработают.

> 🔒 **Про anon-ключ:** он публичный и предназначен для вставки в код сайта — это нормально и безопасно. Доступ ограничен правилами выше (читать отзывы, добавлять отзывы/тикеты). Пароль БД и приватные ключи в код НЕ попадают.

## Как мы будем работать после запуска

- **Новый обзор:** вы говорите «сделай обзор проекта X» → ассистент создаёт страницу + добавляет в реестр → пушит в GitHub → Vercel обновляет сайт.
- **Новость:** «добавь новость про …» → ассистент дописывает в реестр новостей → сайт обновляется.
- **Правки:** «поменяй текст / цвет / фото» → правка в коде → автообновление.
- **Отзывы и тикеты:** работают сами через Supabase, приходят в вашу базу.
